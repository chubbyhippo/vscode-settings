= My VS Code setup

This is my personal VS Code configuration ‚Äî settings, keybindings, and a handful
of extensions I actually use. The centerpiece is modal editing that mirrors my
Emacs `meow` setup, so my hands don't have to change gears when I hop between
editors.

== Modal editing, the meow way (Dance)

If you live in Emacs `meow` (or the IdeaVim setup), this should feel like home.
These settings use the https://github.com/71/dance[Dance] extension, which
brings Kakoune-style *selection-first* editing to VS Code: you select the thing
first, then act on it ‚Äî the opposite of Vim's verb-then-noun. `install-default-extensions.sh`
installs it as `gregoire.dance`.

[NOTE]
====
Dance takes over the editor's `type` command, so it and VSCodeVim can't live
together. If you had VSCodeVim before, see it off first:

[source,shell]
----
code --uninstall-extension vscodevim.vim
----
====

=== It already feels like meow

Most of the muscle memory carries straight over. These match `init.el` with no
help from me:

[cols="1,3"]
|===
| Keys | What they do

| `h` `j` `k` `l` | move (and collapse the selection)
| `H` `J` `K` `L` | extend the selection
| `x`             | select the line (press again to grow it downward)
| `f` `t`         | find / till a character
| `i` `a`         | insert before / after
| `c`             | change
| `y` `p`         | copy / paste
| `u`             | undo
|===

A few keys default to Kakoune rather than meow, so `keybindings.json` remaps them
to match your `init.el` (the last group closes the old gaps ‚Äî see the note below):

[cols="1,2,3"]
|===
| Key | meow | now bound to

| `;` | reverse selection | `dance.selections.changeDirection`
| `g` | cancel selection  | `dance.selections.reduce`
| `s` | kill (cut)        | `dance.edit.yank-delete`
| `d` | delete (no yank)  | `dance.edit.delete`
| `I` | open line above   | `dance.edit.newLine.above.insert`
| `A` | open line below   | `dance.edit.newLine.below.insert`
| `S` | label jump (avy)  | `dance.seek.leap` (Dance's built-in, leap.nvim style)
| `o` `O` | block / to-block (enclosing pair) | `dance.seek.enclosing` / `‚Ä¶extend`
| `'` | repeat last change | `dance.history.repeat.edit`
| `v` | visit (open search) | `dance.search.search`
| `n` `N` | next / previous match | `dance.search.next` / `‚Ä¶previous`
| `,` `.` | inner / around object ‚Äî with meow's thing letters (see note) | `dance.seek.askObject.inner` / `dance.seek.askObject`
| `m` | join lines | `dance.edit.join`
| `r` | replace (paste over selection) | `runCommands`: `dance.edit.delete` + `dance.edit.paste.before`
| `z` | pop selection | `dance.selections.restore`
| `G` | grab (save selection) | `dance.selections.save`
| `w` `W` | mark word / WORD (whole word at point) | `dance.seek.object` inner word / WORD
| `e` `b` | next / previous word (move and select); *with a selection they grow it* word by word, like meow's expandable word selections | word motion + inner-word object; `dance.seek.wordEnd.extend` / `dance.seek.word.extend.backward` when `editorHasSelection`
| `E` `B` | next / previous symbol (WORD); grow when a selection exists | WORD motion + inner-WORD object; `.ws.extend` variants
| `1`‚Äì`9` `0` | meow-expand: with a selection, a digit extends it that many lines down (`0` = 10) ‚Äî the relative gutter numbers are the expand guide, exactly like the IdeaVim setup. Without a selection digits stay Dance counts (`3j`). | `dance.select.line.below.extend` + count
| `-` | pull the expansion back one line | `dance.select.line.above.extend`
| `q` | quit (close) the editor | `workbench.action.closeActiveEditor`
| `X` | goto line (`meow-goto-line`) | `workbench.action.gotoLine`
| `Q` | goto line (`avy-goto-line` stand-in) | `workbench.action.gotoLine`
|===

[NOTE]
====
`,` / `.` open Dance's object picker (inner / around), and `settings.json` now
rewrites that menu (`dance.menus.object`) to meow's `meow-char-thing-table`:
`r` round `s` square `c` curly `g` string `p` paragraph `l` line `b` buffer
`e` symbol `.` sentence ‚Äî plus `q`/`Q` single/grave quotes, `a` angles, `i`
indent, `n` number, `u` argument, space whitespace. The raw pair characters
also work (`,(` `,[` `,{` `,"`), since each entry lists them as alternate keys.
`d`/defun has no VS Code equivalent ‚Äî use `,c` / `.c` for the enclosing braces,
same as the IdeaVim setup. Beginning / end-of-thing stay on Dance's native
`[` / `]`, and they understand the same letters.

Since `,` is no longer "reduce", collapse several cursors back to one with `g`
(cancel selection) or `Esc`.

`r` (replace) is composed as black-hole `delete` + `paste.before`, so it overwrites
the selection with the clipboard without disturbing it. If your Dance version pastes
oddly, swap it for the built-in `dance.edit.yank-replace` (which also copies the old
text to a register).

Digit-expand can't tell word-wise from line-wise selections (VS Code has no
"linewise" context), so digits always extend by *lines*; grow word-wise
selections with `e` / `b` instead ‚Äî that pairing covers both meow expansion
axes.
====

Leave insert mode with `Esc` (the old `jj` trick was a VSCodeVim thing and is
gone). Window focus stays on `ctrl+h/j/k/l` in normal mode.

=== The leader is the Emacs / Meow keypad

The `space` leader now mirrors how Meow's keypad reaches the Emacs keymap in
`init.el`, exactly like the IdeaVim setup: press the same keys after `space`
that you'd press after `SPC` in Emacs. `SPC x f` = `C-x C-f` = find-file,
`SPC c f` = `C-c f`, `SPC m x` = `M-x`. Whatever has no dedicated key is one
`M-x` away: `space m x` (or `alt+x`, or `shift shift`) opens the Command
Palette.

Direct extras, same as the IdeaVim leader: `space enter` ‚âà embark-act ‚Üí
quick fix, `space /` toggles the line comment, `space b b` = consult-buffer
(MRU editors), `space space` = alternate file, and `space s s` is now
consult-line (Find in file; Go to Symbol in Workspace moved to `space s S`).
Two `C-c` keys were repurposed for the keypad: format-document moved off
`space c f` (still on `space f o` and `ctrl+alt+l`), and settings JSON moved
off `space c s` (still on `space . s`).

=== Emacs chords ‚Äî the `C-` / `M-` layer

In the IntelliJ setup every Emacs editing chord is routed to the IDE's Emacs
keymap; the same chords now work here (in insert mode and, where sensible,
everywhere):

* `C-f` / `C-b` / `C-n` / `C-p` ‚Äî char / line motion (`C-n`/`C-p` also walk
  the completion popup and Quick Open lists, like corfu / vertico)
* `C-a` / `C-e` ‚Äî line start / end
* `C-d` ‚Äî delete char, `M-d` ‚Äî delete word, `M-f` / `M-b` ‚Äî word motion (insert mode)
* `C-k` ‚Äî kill to end of line
* `C-w` / `M-w` ‚Äî cut / copy the selection (insert mode for `M-w`), `C-y` ‚Äî yank (paste)
* `C-s` / `C-r` ‚Äî incremental search: open Find, then `C-s` / `C-r` walk the
  matches while the find input has focus
* `C-l` ‚Äî recenter the view on the cursor
* `M-<` / `M->` ‚Äî buffer start / end
* `M-x` ‚Äî Command Palette, `M-o` ‚Äî leap (avy), `M-r` / `M-R` ‚Äî expand / contract
  selection (expreg), `M-.` ‚Äî go to definition (embark-dwim), `M-;` ‚Äî comment,
  `M-%` ‚Äî find & replace
* `C-.` ‚âà embark-act ‚Äî quick fix / code actions (also `alt+enter` and `space enter`)
* `C-z` / `C-S-z` ‚Äî undo / redo, as in `init.el`

Deliberate departures that this replaced (all still reachable elsewhere):
`ctrl+s` no longer saves (auto-save is on; `space x s` / `space c s` save all),
`ctrl+y` no longer deletes lines (`x` `s` kills a line the meow way),
`ctrl+b` no longer goes to definition (use `alt+.`, `f4`, or `space m .`),
`ctrl+a` no longer selects all (use `,b` ‚Äî inner buffer), and in insert mode
`ctrl+k` / `ctrl+l` are kill-line / recenter instead of up / right (`ctrl+p` /
`ctrl+n` / `ctrl+f` cover movement). `ctrl+x` stays cut, so the real `C-x`
*chords* don't exist ‚Äî the `space x` keypad group is the substitute, exactly as
in the IdeaVim setup.

The one leader addition is `space v` ‚Üí Dance's regex *select-within*
(`dance.selections.select`, Kakoune's old `s`) ‚Äî handy for dropping a cursor on
every match in a region. It moved here because `s` is now your meow cut.

=== Multi-cursors

You always have at least one selection, each with its own cursor; "multi-cursor"
just means more than one. Add a cursor on the line below with `C` (above with
`Shift+Alt+C`), split a multi-line selection into one cursor per line with
`Alt+s`, drop a cursor on every regex match with `Alt+k` (filter) or `space v`
(select-within), then collapse back to a single cursor with `g` (or `Esc`).

=== Where it can't quite match meow

The few meow ideas that still have no clean equivalent ‚Äî the honest remaining gaps:

* *Expansion hint overlays* ‚Äî meow paints the numbered targets in the buffer.
  Digit-expand itself works now (select, then press a digit), but the only
  visible guide is the relative line-number gutter, and digits always extend by
  *lines* (grow word-wise with `e` / `b` ‚Äî VS Code has no "linewise selection"
  context to switch on, the trick IdeaVim's `mode() ==# "V"` pulls off).
* *The keypad* ‚Äî meow's `SPC` / `M-SPC` reaches the *entire* Emacs keymap live;
  the `space` leader here mirrors the same `x` / `c` / `m` / `w` groups but is
  still a fixed, hand-curated menu. `M-SPC` from insert mode has no equivalent.
* *`U` undo-in-selection* and the *swap / sync grab* (`R` `Y`) half of the grab
  system. `G` (grab / save) and `z` (pop / restore) work via Dance's selection
  stack, but the swap/sync refinements don't map. `U` is plain redo instead,
  like the IdeaVim setup.
* *`M-y` / consult-yank-pop* ‚Äî VS Code has no clipboard ring; install a
  clipboard-history extension if you miss it.
* *which-key* ‚Äî the leader menu shows nothing while you type. The
  `VSpaceCode.whichkey` extension can fill that hole, but it's a different
  leader mechanism, so it's not wired in here.

Everything else now carries over ‚Äî `o` / `O` block (enclosing pair), `m` join,
`,` / `.` inner / around objects with meow's thing letters, digit expand,
word-grow on `e` / `b`, `z` pop-selection, `G` grab, `r` replace, `q` quit,
`X` goto-line, `v` `n` `N` search (visit), `'` repeat, and the full
`SPC x` / `SPC c` / `SPC m` / `SPC w` keypad ‚Äî see the tables above.

== Getting set up

There are two things to pull in: my settings + keybindings, and the extensions.
Grab whichever line matches your shell ‚Äî they all just download the files into
your VS Code config and run the `code` CLI for you.

=== Settings & keybindings

This drops `settings.json` and `keybindings.json` into place.

[source,shell,subs="attributes"]
.curl
----
curl {raw_base_settings_url}/install.sh | /usr/bin/env sh
----

[source,shell,subs="attributes"]
.wget
----
wget -qO - {raw_base_settings_url}/install.sh | /usr/bin/env sh
----

[source,powershell,subs="attributes"]
.PowerShell
----
irm {raw_base_settings_url}/install.ps1 | iex
----

== Starting fresh

Want a clean slate before reinstalling? This pulls out *every* extension you have.
It runs the sweep twice on purpose ‚Äî the second pass clears anything that was
only hanging on as another extension's dependency.

[source,shell]
.sh
----
code --list-extensions | xargs -L 1 code --uninstall-extension;code --list-extensions | xargs -L 1 code --uninstall-extension
----

[source,powershell]
.PowerShell
----
code --list-extensions | foreach { code --uninstall-extension $_ };code --list-extensions | foreach { code --uninstall-extension $_ }
----
