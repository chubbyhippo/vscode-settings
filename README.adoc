= My VS Code setup

This is my personal VS Code configuration ‚Äî settings, keybindings, and a handful
of extensions I actually use. The centerpiece is modal editing that mirrors my
Emacs `meow` setup, so my hands don't have to change gears when I hop between
editors.

== Modal editing, the meow way (Dance)

If you live in Emacs `meow` (or the IdeaVim setup), this should feel like home.
These settings use the https://github.com/71/dance[Dance] extension, which
brings Kakoune-style *selection-first* editing to VS Code: you select the thing
first, then act on it ‚Äî the opposite of Vim's verb-then-noun. `install-default-extensions.sh`
installs it as `gregoire.dance`.

[NOTE]
====
Dance takes over the editor's `type` command, so it and VSCodeVim can't live
together. If you had VSCodeVim before, see it off first:

[source,shell]
----
code --uninstall-extension vscodevim.vim
----
====

=== It already feels like meow

Most of the muscle memory carries straight over. These match `init.el` with no
help from me:

[cols="1,3"]
|===
| Keys | What they do

| `h` `j` `k` `l` | move (and collapse the selection)
| `H` `J` `K` `L` | extend the selection
| `x`             | select the line (press again to grow it downward)
| `f` `t`         | find / till a character
| `i` `a`         | insert before / after
| `c`             | change
| `y` `p`         | copy / paste
| `u`             | undo
|===

A few keys default to Kakoune rather than meow, so `keybindings.json` remaps them
to match your `init.el` (the last group closes the old gaps ‚Äî see the note below):

[cols="1,2,3"]
|===
| Key | meow | now bound to

| `;` | reverse selection | `dance.selections.changeDirection`
| `g` | cancel selection  | `dance.selections.reduce`
| `s` | kill (cut)        | `dance.edit.yank-delete`
| `d` | delete (no yank)  | `dance.edit.delete`
| `I` | open line above   | `dance.edit.newLine.above.insert`
| `A` | open line below   | `dance.edit.newLine.below.insert`
| `S` | label jump (avy)  | `dance.seek.leap` (Dance's built-in, leap.nvim style)
| `o` `O` | block / to-block (enclosing pair) | `dance.seek.enclosing` / `‚Ä¶extend`
| `'` | repeat last change | `dance.history.repeat.edit`
| `v` | visit (open search) | `dance.search.search`
| `n` `N` | next / previous match | `dance.search.next` / `‚Ä¶previous`
| `,` `.` | inner / around object | `dance.seek.askObject.inner` / `dance.seek.askObject`
| `m` | join lines | `dance.edit.join`
| `r` | replace (paste over selection) | `runCommands`: `dance.edit.delete` + `dance.edit.paste.before`
| `z` | pop selection | `dance.selections.restore`
| `G` | grab (save selection) | `dance.selections.save`
| `w` `W` | mark word / WORD (whole word at point) | `dance.seek.object` inner word / WORD
| `e` `b` | next / previous word (word only, no gap) | word motion + inner-word object
|===

[NOTE]
====
`,` / `.` now open Dance's object picker (inner / around); the *follow-up* key is
Dance's own object key (a bracket, quote, or letter ‚Äî e.g. `(` `{` `[` `"` `w` `p`),
not meow's `r s c g p l b` thing-letters ‚Äî so it's close to meow, not identical.
Since `,` is no longer "reduce", collapse several cursors back to one with `g`
(cancel selection) or `Esc`. Beginning / end-of-thing stay on Dance's native `[` / `]`.

`r` (replace) is composed as black-hole `delete` + `paste.before`, so it overwrites
the selection with the clipboard without disturbing it. If your Dance version pastes
oddly, swap it for the built-in `dance.edit.yank-replace` (which also copies the old
text to a register).
====

Your `space` leader and the `ctrl+...` IntelliJ layer are untouched ‚Äî they're
just gated on `dance.mode == 'normal'` now instead of Vim's normal mode. Window
focus stays on `ctrl+h/j/k/l`. Leave insert mode with `Esc` (the old `jj` trick
was a VSCodeVim thing and is gone).

The one leader addition is `space v` ‚Üí Dance's regex *select-within*
(`dance.selections.select`, Kakoune's old `s`) ‚Äî handy for dropping a cursor on
every match in a region. It moved here because `s` is now your meow cut.

=== Multi-cursors

You always have at least one selection, each with its own cursor; "multi-cursor"
just means more than one. Add a cursor on the line below with `C` (above with
`Shift+Alt+C`), split a multi-line selection into one cursor per line with
`Alt+s`, drop a cursor on every regex match with `Alt+k` (filter) or `space v`
(select-within), then collapse back to a single cursor with `g` (or `Esc`).

=== Where it can't quite match meow

A few meow ideas still have no clean Dance equivalent ‚Äî the honest remaining gaps:

* *Expansion hints* (`1`‚Äì`9`, `0`) ‚Äî meow's numbered selection-expand. Dance
  reads digits as repeat counts instead.
* *The keypad* ‚Äî meow's `SPC` / `M-SPC` reaches the *entire* Emacs keymap live;
  the `space` leader here is a fixed, hand-curated menu.
* *`U` undo-in-selection* and the *swap / sync grab* (`R` `Y`) half of the grab
  system. `G` (grab / save) and `z` (pop / restore) now work via Dance's
  selection stack, but the swap/sync refinements don't map.
* *Word grow-arming* ‚Äî in meow, `w` then `e`/`b` *grows* the selection word by
  word. Here `w`/`W` grab the word cleanly and `e`/`b` select the next/previous
  word, but they don't extend an existing `w` selection (armed-grow needs the kind
  of stateful script that Dance can't run).

Everything else now carries over ‚Äî `o` / `O` block (enclosing pair), `m` join,
`,` / `.` inner / around objects, `z` pop-selection, `G` grab, `r` replace, and
`v` `n` `N` search (visit) plus `'` repeat ‚Äî see the remap table above.

== Getting set up

There are two things to pull in: my settings + keybindings, and the extensions.
Grab whichever line matches your shell ‚Äî they all just download the files into
your VS Code config and run the `code` CLI for you.

=== Settings & keybindings

This drops `settings.json` and `keybindings.json` into place.

[source,shell,subs="attributes"]
.curl
----
curl {raw_base_settings_url}/install.sh | /usr/bin/env sh
----

[source,shell,subs="attributes"]
.wget
----
wget -qO - {raw_base_settings_url}/install.sh | /usr/bin/env sh
----

[source,powershell,subs="attributes"]
.PowerShell
----
irm {raw_base_settings_url}/install.ps1 | iex
----

=== Extensions

The default pack: Dance, the GitHub theme, Material icons, the Git tools,
SonarLint, Prettier, and the rest of my everyday kit.

[source,shell,subs="attributes"]
.curl
----
curl {raw_base_settings_url}/install-default-extensions.sh | /usr/bin/env sh
----

[source,powershell,subs="attributes"]
.PowerShell
----
Invoke-RestMethod {raw_base_settings_url}/install-default-extensions.sh | Invoke-Expression
----

Working in a particular language? There are smaller, per-language packs (Clojure,
Go, Java, JavaScript, Python, Rust) in the
link:https://github.com/chubbyhippo/vscode-settings/tree/main/extensions[extensions
folder] ‚Äî install only the ones you need.

== Starting fresh

Want a clean slate before reinstalling? This pulls out *every* extension you have.
It runs the sweep twice on purpose ‚Äî the second pass clears anything that was
only hanging on as another extension's dependency.

[source,shell]
.sh
----
code --list-extensions | xargs -L 1 code --uninstall-extension;code --list-extensions | xargs -L 1 code --uninstall-extension
----

[source,powershell]
.PowerShell
----
code --list-extensions | foreach { code --uninstall-extension $_ };code --list-extensions | foreach { code --uninstall-extension $_ }
----
